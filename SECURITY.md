# Security Policy

## Reporting

Do not open public issues for vulnerabilities, leaked-sensitive-data findings,
credentials, private account material, or abuse-enabling bypass details.

Report security issues through GitHub private vulnerability reporting for this
repository. If that is unavailable, contact the repository owner privately before
sharing details.

## Supported Version

`v0.1.0` is the first public source release. Security fixes target `main` first;
release tags are updated only by publishing a new release.

## Responsible Use

Use synthetic, non-sensitive research topics for tests and examples. Do not put
secrets, personal data, private customer data, private browser profiles, or
credential material into `askq`, `idr`, NotebookLM, generated reports, issues, or
pull requests.

Do not use this project to violate platform terms, privacy law, anti-abuse
controls, or consent boundaries.
